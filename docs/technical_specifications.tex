\section{Executive Summary}
%=============================================================================

This document specifies the technical design of a \textbf{Smart City Traffic Management System} that employs multi-agent reinforcement learning (RL) and multi-criteria decision making (MCDM) to optimize traffic signal control across a city-scale grid. The system models 16 intersections as independent RL agents that learn adaptive signal timing policies to reduce congestion, emissions, and travel time while maintaining safety and public-transport priority. A React.js dashboard provides real-time visualization and comparative analysis of RL policies evaluated through WSM and TOPSIS methods.

%=============================================================================
\section{Problem Statement}
%=============================================================================

Urban traffic management faces the challenge of coordinating traffic lights, autonomous vehicles, and public transport agents to simultaneously:
\begin{itemize}[nosep]
    \item Minimize average vehicle wait time and queue lengths
    \item Maximize intersection throughput
    \item Reduce CO$_2$ emissions from idling vehicles
    \item Ensure safety (minimize near-miss incidents)
    \item Provide signal priority for public buses and emergency vehicles
\end{itemize}

Traditional fixed-timer signal control cannot adapt to dynamic traffic patterns. This system applies \textbf{Q-Learning} and \textbf{SARSA} algorithms to learn adaptive policies that respond in real time to changing traffic conditions across five distinct scenarios.

\section{Environment Specification}
%=============================================================================

\subsection{Grid Layout}

The environment consists of a \textbf{4$\times$4 grid} of 16 signalized intersections. Each intersection is named after a real Dubai location (e.g., Deira City Centre, Dubai Mall Junction, Marina Walk).

\subsection{State Space}

Each intersection's state is a tuple $(q_{NS}, q_{EW}, \phi)$ where:
\begin{itemize}[nosep]
    \item $q_{NS} \in \{0, 1, 2, 3, 4\}$ --- discretized North--South queue length (bins of 5 vehicles)
    \item $q_{EW} \in \{0, 1, 2, 3, 4\}$ --- discretized East--West queue length
    \item $\phi \in \{0, 1\}$ --- current signal phase (0 = NS green, 1 = EW green)
\end{itemize}

Total state space per intersection: $5 \times 5 \times 2 = 50$ states.

\subsection{Action Space}

Each agent selects one of three actions at each time step:
\begin{enumerate}[nosep, start=0]
    \item \textbf{KEEP} --- Maintain current signal phase
    \item \textbf{SWITCH} --- Initiate yellow phase and switch to opposite phase
    \item \textbf{EXTEND} --- Extend current green phase by one additional step
\end{enumerate}

Constraints: minimum green time of 10 steps before switching is allowed; yellow phase duration is 3 steps.

\subsection{Traffic Dynamics}

\begin{itemize}[nosep]
    \item \textbf{Arrivals}: Modeled as a Poisson process calibrated to a 24-hour Dubai traffic pattern with base volume per intersection (400--1160 vehicles/hour).
    \item \textbf{Departures}: Green phase serves 2--5 vehicles per time step (uniformly sampled).
    \item \textbf{Queue cap}: Maximum 100 vehicles per direction per intersection.
    \item \textbf{Time mapping}: Each step represents approximately 30 seconds of real time.
    \item \textbf{Hour progression}: Simulation hour advances as $h = 8 + \lfloor \text{step} \times 30 / 3600 \rfloor \mod 24$.
\end{itemize}

\subsection{Reward Function}

The reward for intersection $i$ at step $t$ is:
\begin{equation}
    R_i^t = -0.1(q_{NS} + q_{EW}) + 0.05 \cdot V_{\text{passed}} - 0.5 \cdot E - 2.0 \cdot M - 0.05 \cdot |q_{NS} - q_{EW}|
\end{equation}
where:
\begin{itemize}[nosep]
    \item $q_{NS}, q_{EW}$ are current queue lengths (penalty for waiting vehicles)
    \item $V_{\text{passed}}$ is cumulative vehicles passed (throughput reward)
    \item $E$ is cumulative CO$_2$ emissions (kg), computed as $0.002 \times \text{idle vehicles}$ per step
    \item $M$ is count of near-miss incidents (safety penalty)
    \item $|q_{NS} - q_{EW}|$ is queue imbalance (balance penalty)
\end{itemize}

\section{Reinforcement Learning Agents}
%=============================================================================

\subsection{Q-Learning Agent (Off-Policy TD Control)}

\begin{itemize}[nosep]
    \item \textbf{Update rule}: $Q(s,a) \leftarrow Q(s,a) + \alpha \big[ r + \gamma \max_{a'} Q(s', a') - Q(s,a) \big]$
    \item \textbf{Exploration}: $\varepsilon$-greedy with decay: $\varepsilon \leftarrow \max(\varepsilon_{\min}, \varepsilon \times 0.995)$
    \item \textbf{Hyperparameters}: $\alpha = 0.1$, $\gamma = 0.95$, $\varepsilon_0 = 1.0$, $\varepsilon_{\min} = 0.01$
\end{itemize}

\subsection{SARSA Agent (On-Policy TD Control)}

\begin{itemize}[nosep]
    \item \textbf{Update rule}: $Q(s,a) \leftarrow Q(s,a) + \alpha \big[ r + \gamma Q(s', a') - Q(s,a) \big]$
    \item Same hyperparameters as Q-Learning
    \item \textbf{Key difference}: Uses the actual next action $a'$ (chosen by the same policy) rather than $\max_{a'} Q(s', a')$
\end{itemize}

\subsection{Fixed-Timer Baseline}

\begin{itemize}[nosep]
    \item No learning; switches signal phase every 15 steps deterministically
    \item Serves as the non-adaptive baseline for comparison
\end{itemize}

\subsection{Multi-Agent Controller}

Each of the 16 intersections receives its own independent agent instance. A \texttt{MultiAgentController} manages the collection, dispatching states, collecting actions, and coordinating updates. This represents a \textbf{decentralized} multi-agent architecture where agents learn independently but interact through shared traffic flows.

\section{Multi-Criteria Decision Making (MCDM)}
%=============================================================================

\subsection{Decision Matrix}

Alternatives (traffic control policies) are evaluated across four criteria:

\begin{longtable}{p{4cm}p{2.5cm}p{5.5cm}}
    \toprule
    \textbf{Criterion} & \textbf{Direction} & \textbf{Description} \\
    \midrule
    Efficiency (Throughput) & Maximize & Total vehicles passed through the network \\
    Safety Score (0--100) & Maximize & Composite score; penalizes near-misses and hard stops \\
    CO$_2$ Emissions (kg) & Minimize & Total idling emissions across all intersections \\
    Public Transport Delay & Minimize & Average bus delay at signalized intersections \\
    \bottomrule
\end{longtable}

\textbf{Default weights}: $w = [0.35, 0.25, 0.25, 0.15]$ (efficiency, safety, emissions, public transport).

\subsection{Weighted Sum Model (WSM)}

\begin{equation}
    S_i^{\text{WSM}} = \sum_{j=1}^{n} w_j \cdot \bar{r}_{ij}
\end{equation}
where $\bar{r}_{ij}$ is the min--max normalized value for alternative $i$ on criterion $j$.

\subsection{TOPSIS}

\begin{enumerate}[nosep]
    \item \textbf{Vector normalize}: $\bar{r}_{ij} = r_{ij} / \sqrt{\sum_k r_{kj}^2}$
    \item \textbf{Weight}: $v_{ij} = w_j \cdot \bar{r}_{ij}$
    \item \textbf{Ideal solution}: $A^+ = (\max_i v_{ij})$ for benefit criteria, $(\min_i v_{ij})$ for cost criteria
    \item \textbf{Anti-ideal}: $A^- = (\min_i v_{ij})$ for benefit, $(\max_i v_{ij})$ for cost
    \item \textbf{Distances}: $D_i^+ = \sqrt{\sum_j (v_{ij} - A_j^+)^2}$, \; $D_i^- = \sqrt{\sum_j (v_{ij} - A_j^-)^2}$
    \item \textbf{Closeness}: $C_i = D_i^- / (D_i^+ + D_i^-)$
\end{enumerate}

\subsection{Sensitivity Analysis}

Criteria weights are varied by $\pm 0.15$ in 5 steps per criterion. The remaining weight delta is redistributed uniformly across other criteria. Rankings from both WSM and TOPSIS are recorded at each variation to assess robustness.

\section{Non-Functional Requirements}
%=============================================================================

\begin{itemize}[nosep]
    \item \textbf{Performance}: Full 15-experiment training completes in under 5 minutes on a standard laptop.
    \item \textbf{Reproducibility}: Fixed random seed (42) ensures identical results across runs.
    \item \textbf{Portability}: Python 3.9+ with pip-installable dependencies; Node.js 18+ for dashboard.
    \item \textbf{Scalability}: Grid size and agent count are parameterized for future expansion.
